# Documentation UX Improvements

A planned improvement pass on the in-app documentation experience — structure, navigation, and first-time readability.

---

## What This Covers

- **Setup Guide restructure** — step-by-step sections with clear bullet points instead of dense paragraphs; "Drop LBM into Any Project" flow cleaned up
- **Non-developer path first** — installation section explicitly ordered so the non-technical path always leads; headings renamed to "For Everyone" / "For Developers" for clarity
- **Cross-linking About ↔ Setup Guide** — both pages carry a visible callout linking to the other so users always have a quick path to context or action
- **Docs front-end polish** — blockquotes, callouts, and bullet lists styled consistently across all doc pages in the viewer

---

## Status

Parked for a future sprint. Some items (cross-linking, Setup Guide restructure) have already shipped. The remaining work is styling and consistency polish across all doc pages.
